## Project Overview

Nebula Dreamcast Engine is a 3D game engine + editor targeting **Windows (desktop editor)** and **Sega Dreamcast (runtime)**. The workflow is: edit in the desktop editor, package for Dreamcast, test on emulator/hardware.

## Build Commands

### Windows Editor (CMake + Visual Studio)
```bat
cmake -S . -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Debug
```
Run: `./build/Debug/NebulaEditor.exe`

### With Dreamcast SDK wiring
```bat
cmake -S . -B build -DNEBULA_USE_DREAMCAST_SDK=ON -DKOS_BASE="C:/DreamSDK/opt/toolchains/dc/kos" -DKOS_CC_BASE="C:/DreamSDK/opt/toolchains/dc"
```

### Dreamcast package flow
1. Export/package from editor
2. Run `_nebula_build_dreamcast.bat` in `build_dreamcast/`
3. Test `nebula_dreamcast.cdi` on emulator/hardware

There is no unit test target. Validate changes by building and exercising flows in the editor.

## Architecture

### Editor monolith: `src/main.cpp` (~841KB)
The editor is a single large ImGui+OpenGL2 application. `main.cpp` contains the bulk of editor logic: viewport rendering, scene editing UI, asset import/export (via Assimp), Dreamcast code generation, texture conversion, mesh inspection, animation baking, and project management. This is the primary file you'll modify for editor features.

### Three-layer Dreamcast runtime architecture
1. **Script layer** (`Scripts/*.c`) — Gameplay scripts implementing `NB_Game_OnStart`, `NB_Game_OnUpdate`, `NB_Game_OnSceneSwitch`. Call `NB_RT_*` bridge functions.
2. **Generated runtime layer** (`build_dreamcast/main.c`) — Generated by the editor from `src/main.cpp`. Implements `NB_RT_*` bridge functions, calls `NB_DC_*` and `NB_KOS_*` APIs.
3. **Platform bindings layer** (`src/platform/dreamcast/`) — `KosBindings.c/.h` (scene/mesh/texture loading), `KosInput.c/.h` (controller input wrappers).

### Key modules
- **`src/scene/`** — Scene serialization (`SceneIO`), node type definitions (`NodeTypes.h`): `StaticMesh3DNode`, `Camera3DNode`, `Node3DNode`, `Audio3DNode`. Scene hierarchy with parent chains and cycle detection (`NebulaNodes` namespace).
- **`src/assets/meta_io`** — Material (.nebmat), texture metadata, and slot manifest (.nebslots) I/O (`NebulaAssets` namespace).
- **`src/camera/Camera3D`** — Runtime camera logic.
- **`src/editor/camera3d_v2`** — Editor viewport camera (v2).
- **`src/audio3d`** — Audio3D node support.
- **`src/platform/dreamcast/build_helpers`** — Disc image detection and build utilities.
- **`src/nebula_dreamcast`** — Dreamcast SDK availability check.

### Custom file formats
- `.nebproj` — project, `.nebscene` — scene, `.nebmesh` — mesh, `.nebtex` — texture (16-bit BE), `.nebmat` — material, `.nebslots` — material-slot manifest, `.nebanim` — vertex animation clip.

### Asset staging for Dreamcast
Deterministic short names under `build_dreamcast/cd_root/data/`: Scenes `Sxxxxx`, Meshes `Mxxxxx`, Textures `Txxxxx`, Animations `Axxxxx`. Missing textures use a fallback white texture at runtime.

### Dependencies
- **Vendored** in `thirdparty/`: GLFW, Dear ImGui (OpenGL2 + GLFW backends)
- **Fetched** via CMake FetchContent: Assimp v5.3.1
- **Windows**: links opengl32, gdi32, user32, shell32, winmm, windowscodecs

## Coding Conventions

- **C++17** (set in CMakeLists.txt)
- **4-space indentation**, braces on next line for functions/types
- **PascalCase** for types/functions, **camelCase** for locals/fields, **kPrefix** for constants
- Gameplay scripts are plain **C** (not C++), use `NB_SCRIPT_EXPORT` macro for DLL exports on Windows
- Commit style: scoped imperative summaries, e.g. `Dreamcast: fix NPOT parser crash`

## Important Patterns

- `NEBULA_USE_DREAMCAST_SDK` CMake option controls whether KOS headers/libs are wired in (OFF by default for desktop-only builds)
- Editor preferences (DreamSDK path, MSVC path) persist in `editor_prefs.ini`
- The editor generates `build_dreamcast/main.c` and the Dreamcast Makefile — do not hand-edit generated output
- Material slots: up to 14 per StaticMesh (`kStaticMeshMaterialSlots`)
- Scene tokens are encoded/decoded for serialization safety (`EncodeSceneToken`/`DecodeSceneToken`)
